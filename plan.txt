# MBA Partner — Courses Page Redesign
## Build Spec / Prompt for Code Generation

> Paste this whole file as the prompt. Build a single, premium, conversion-optimized
> Courses page for **MBA Partner** — a post-admission, pre-placement career-acceleration
> community for current MBA students (NOT an admissions consulting company). It sells
> Live Projects, Case Competitions, a Placements Bootcamp, bundled combos, and
> certification add-ons that help students land Summer Internship Placements (SIP) and
> Final Placements at top recruiters.

---

## 1. Brand Voice & Aesthetic Direction

Build this like **"Harvard Business School meets McKinsey meets modern SaaS."**
Premium, restrained, institutional — not a cheerful edtech marketplace.

**Hard rules — do not violate:**
- No gradients, anywhere, on anything.
- No pill-shaped buttons or blob/organic background shapes.
- No doodle illustrations, no graduation-cap/lightbulb iconography, no emoji-style icon badges.
- No stock photos of generic smiling students giving thumbs-up.
- Flat fills only. Sharp, restrained geometry (10px max corner radius on cards).
- One accent color (gold/brass) used in **under 5% of any viewport** — reserve it for
  primary buttons, numeral highlights, and a single "Most Chosen" tag. Never as a section background.
- Numbers are always sourced/specific ("98.7% placed in desired domain") — never vague ("99% satisfaction").

## 2. Design Tokens

### Color
```css
--navy:           #0B1B33;  /* primary brand, headlines, dark section bg, nav text */
--navy-body:       #13202E; /* default body text on light bg */
--ivory:          #FAF8F3;  /* default page background — NOT pure white */
--white:          #FFFFFF;  /* card surfaces */
--panel:          #F4F1E9;  /* alternate section background */
--slate:          #5B6B7F;  /* secondary/muted text, captions */
--hairline:       #D8DEE6;  /* all borders, dividers, table grid lines — 1px only */

--gold:           #C8962C;  /* PRIMARY ACCENT — CTA fill, numeral highlights, active states */
--gold-hover:     #A87A1E;  /* button hover/active */
--gold-tint:      #F4E9D8;  /* light wash behind small tags only */

--success:        #1E5C4A;  /* reserved ONLY for "Verified Outcome" tags / checkmarks */
```
Usage ratio: **~80% neutral (ivory/white/slate), ~15% navy, ~5% gold.** Exactly **two**
full-bleed dark-navy sections on the whole page (Metrics Band + Final CTA) — no more.

### Typography
- **Display/Headings:** Playfair Display (Google Fonts) — serif, institutional weight.
- **Body/UI:** Inter (Google Fonts) — clean, fast-scanning sans.
- Numerals in stat tiles use `font-variant-numeric: tabular-nums`.

| Token | Desktop | Mobile | Line-height | Weight |
|---|---|---|---|---|
| H1 / Hero | 56px | 34px | 1.1 | 600 |
| H2 | 40px | 28px | 1.15 | 600 |
| H3 | 28px | 22px | 1.2 | 600 |
| H4 | 20px | 18px | 1.3 | 600 |
| Eyebrow label | 13px | 12px | 1.4 | 600, uppercase, +0.08em tracking |
| Body large | 18px | — | 1.6 | 400 |
| Body default | 16px | — | 1.6 | 400 |
| Body small / meta | 14px | — | 1.5 | 400–500 |
| Stat numeral | 48px | 32px | 1.1 | 600, tabular-nums |
| Button label | 15px | — | 1.0 | 600, +0.02em tracking |

### Spacing (8px base unit)
`4 / 8 / 16 / 24 / 32 / 48 / 80 / 120` px scale. Section vertical padding = 80px desktop
(120px for Hero & Final CTA), halved on mobile. Card padding = 32px desktop / 24px mobile.

### Components
- **Card:** white surface, 1px `--hairline` border, 10px radius, no shadow at rest,
  hover → `translateY(-2px)` + `box-shadow: 0 12px 24px rgba(11,27,51,0.08)`, 150–200ms ease.
- **Primary button:** solid `--gold` fill, `--ivory` text, 8px radius, 48px height
  (44px mobile min), hover → `--gold-hover`, no shadow.
- **Secondary button:** transparent, 1px `--navy` border (or white border on dark sections),
  text matches border, hover → 8% navy tint fill.
- **Text link:** no border/fill, navy text, gold underline on hover.
- **Tag — "Most Chosen":** `--gold-tint` bg, `--gold-hover` text, 4px radius, 11px uppercase.
- **Tag — domain pill:** outlined, 1px navy @30% opacity, 999px radius (the only approved pill shape).
- **Icons:** line-style only (Lucide/Feather, 1.5px stroke), navy or slate color — never gold, never filled/3D.

## 3. Page Sections (build in this exact order)

### 01 — Sticky Nav Bar
Wordmark + small-caps credibility line ("Initiative by Alumni of Old IIM") on the left.
Nav links: Home / Programs / Mentors / Testimonials. Single solid gold primary button
on the right: **"Enroll Now"**. Transparent/ivory at top → white bg + soft shadow after ~80px scroll.

### 02 — Hero
Asymmetric ~58/42 split.
- Left: eyebrow label ("For MBA Students Heading Into Placements"), serif H1
  (e.g. *"Committee Roles Don't Get You Shortlisted. Outcomes Do."*), one-line subhead
  naming the mechanism (live projects, case comps, bootcamp), inline stat strip
  (**9.6/10 rating · 5,000+ network · 98.7% placed in desired domain**), two CTAs:
  primary solid **"Explore Programs"**, secondary ghost **"Talk to a Mentor"**.
- Right: one confident photo (mentor + student, real setting — not stock thumbs-up),
  with a small white floating proof card pinned at lower corner (1 stat only, e.g.
  "700+ students mentored").

### 03 — Recruiter Trust Bar
Slim full-width band, pale-navy tint. Label: "Our Students Now Work At". Single-row
grayscale logo strip (7–9 logos: Accenture, KPMG, Kearney, Reliance, Pine Labs,
JM Financial, Times of India, Media.net). Logos at 60% opacity/grayscale at rest →
full color on hover. On mobile: horizontal swipeable strip with scroll-snap.

### 04 — Outcome Metrics Band (dark navy, full-bleed)
4-tile grid (2×2 mobile). Each tile: large gold-accented tabular numeral + small-caps
label beneath, thin gold top-rule.
Tiles: **9.6/10** Average Mentor Rating · **5,000+** Student Network ·
**98.7%** Placed In Desired Domain · **40+** Mentors From Top B-Schools.
Numerals count up from 0 on scroll-into-view (respect `prefers-reduced-motion`).

### 05 — Guided Path Finder
Heading: "Where are you starting from?" Three equal-weight bordered cards:
- A: "I have no real project on my CV" → links to Live Projects
- B: "I freeze in placement interviews" → links to Placements Bootcamp
- C: "I want the complete package" → links to flagship combo

Each: short diagnostic copy + one benefit line + text link (not a full button).

### 06 — Featured Programs Grid
Section intro ("Programs Built Around Placement Season"). 3-up card grid (1-up mobile).
**Fixed card anatomy, same order every card:**
category tag → program name → one-line outcome description → meta row
(duration · format · domains) → price (label as "Cohort Price", strike-through only if
a real discount applies) → full-width primary button "View Program".
The flagship all-in-one combo gets a hairline **gold border** + small **"Most Chosen"** tag —
no louder color, just the border + tag.

Products to map onto cards (from current catalog):
- All-In-One Combo (Bootcamp + Case Comp + Live Project) — flagship, "Most Chosen"
- Placements Bootcamp (+ combo variants: +Case Comp / +Live Project)
- Live Projects (1 domain / 2 domains)
- Case Competition (+ Live Project combo)
- Certifications: Advanced Excel, Power BI

### 07 — Program Comparison Table
Combos as columns, attributes as rows (Live Project access / Case Comp access /
Bootcamp modules / Mentor 1:1 sessions / Domain coverage / Certificate / Price).
Checkmarks for binary attributes. Flagship column gets a subtle gold top-border highlight.
Minimal hairline grid lines, generous cell padding — read as a finance term sheet, not
a marketing table. On mobile: first column sticky/frozen, rest scrolls horizontally.

### 08 — How It Works
Horizontal 4-step timeline (vertical stack mobile), connected by a thin line:
1. Enroll & Onboard → 2. Get Matched to a Mentor & Domain →
3. Execute Live Project / Bootcamp Modules → 4. Mock Interviews & Placement Support.
Large gold tabular numeral per step, short title, one-sentence description. No illustration.

### 09 — Mentor Credibility Wall
4-up grid (2-up tablet, 1-up mobile). Each card: square/4:5 headshot, one-line credential
("IIM Ahmedabad · ex-Strategy, Accenture"), outlined domain pill tag, name set smaller
than the credential line (institution does more persuasive work than the name here).

### 10 — Student Outcome Stories
One large featured pull-quote (navy bg, white serif quote, gold rule) with an
**outcome badge above the quote** (role + company, e.g. "Product Management → Media.net"),
photo, name, institution. Below: 3-up grid of shorter testimonial cards in the same
outcome-badge-first anatomy. Use real testimonials already on the live site
(Nishant – IIM A, Shen Shaji – IIM B → Media.net PM, Rutuja – IIM C → Accenture Strategy, etc.).

### 11 — FAQ
Two-column accordion (desktop), single column (mobile). 6–8 questions:
time commitment alongside coursework, switching domains after enrolling, placement
guarantee (if any), Live Project vs. Case Competition difference, 1:1 mentor access,
what happens after payment. Plus/minus or chevron toggle, generous answer line-height.

### 12 — Final CTA Band (dark navy, full-bleed — visual bookend to Hero)
Headline ("Placement Season Doesn't Wait. Neither Should You."), both CTAs again
(primary "Enroll Now", secondary ghost "Book a Free Career Call"), plus a quiet
reassurance line beneath (refund/guarantee policy if one exists, or
"Trusted by 5,000+ MBA students"). Mobile: CTAs stack full-width, primary on top.

### 13 — Footer
Wordmark + one-line credibility statement, quick links, programs, community channels
(WhatsApp, Telegram, LinkedIn, Instagram), contact, legal. Quiet by design — smallest
type on the page, no competing CTA buttons.

### Mobile-only addition
Persistent slim bottom CTA bar ("Enroll Now") appears after the Hero is scrolled past —
this audience is majority mobile and the current live page has no persistent mobile CTA.

## 4. CTA Hierarchy (do not deviate)
- **Primary** (solid gold): "Enroll Now" / "View Program" — nav, hero, every program card, final CTA.
- **Secondary** (ghost): "Talk to a Mentor" / "Book a Free Career Call" — hero + final CTA only.
  This exists for undecided buyers comparing 5 similar combos — route them to a conversation, not a bounce.
- **Tertiary** (text link): "Join WhatsApp Community" / "Join Telegram" — footer + optional FAQ link only.

---

## 5. Accessibility
- Navy-on-ivory body text exceeds WCAG AA. Gold-on-ivory: large text/UI only, never small body copy.
- All interactive elements ≥44×44px tap target.
- Visible focus states: 2px gold outline, 2px offset — never suppressed.
- Real `<table>` markup with `scope` on headers for the comparison table (not div-grids).
- Descriptive alt text on all mentor/testimonial photos (name + credential).
- Respect `prefers-reduced-motion` on the stat-counter animation.

---

## 6. What NOT to build
- No SKU-list-style flat product grid with no narrative (this is the exact failure of the current live page).
- No countdown timers / fake urgency / "X seats left" scarcity messaging.
- No separate "Admission Roadmaps" or "University Partners" sections — this business is
  post-admission/placement-focused, not an admissions consultancy.
- No more than two full-bleed dark sections total.
